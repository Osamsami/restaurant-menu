# Contributing Guidelines

This project is a solo assignment for Codoc IT Ltd.'s Intern Development Programme. As such, external contributions are not currently being accepted. However, the project adheres to the following internal development standards:

## Branch Naming Convention
Branches are prefixed according to the nature of the work being done:
- `feat/`: For new features or significant additions
- `docs/`: For documentation updates
- `ci/`: For continuous integration or deployment updates
- `fix/`: For bug fixes

## Commit Message Style
Commit messages follow a conventional format using the following prefixes:
- `feat:` for new features
- `docs:` for documentation changes
- `ci:` for CI/CD changes
- `fix:` for bug fixes

## Pull Requests
As this is a solo assignment project, pull requests are self-reviewed by the author before merging into the main branch to ensure code quality and consistency with project standards.
